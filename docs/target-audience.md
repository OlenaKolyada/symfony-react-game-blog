# Target Audience

The target audience of Grem:

1. Players who want a simple gaming portal with games, news, and reviews.
2. Readers who want structured gaming content grouped by tags, genres, platforms, developers, and publishers.
3. Editors and administrators who manage gaming content through a back office.
4. Academic or portfolio reviewers who evaluate the full-stack implementation.

## Hypothetical Portraits

### Gaming Reader

1. Characteristics:
   - Age: 16-40
   - Interested in games, releases, reviews, and gaming news
   - Wants fast browsing by content type and game metadata
2. Technological literacy:
   - Comfortable using web applications
   - Expects clear navigation and readable article pages
   - Uses desktop or mobile browser

### Registered Community User

1. Characteristics:
   - Has an account in the portal
   - Wants access to personal profile data
   - May comment on reviews when comment functionality is enabled in the UI
2. Technological literacy:
   - Familiar with email and password authentication
   - Expects persistent login while using the portal

### Content Administrator

1. Characteristics:
   - Creates and edits games, news, reviews, tags, genres, platforms, developers, publishers, comments, and users
   - Needs a back office for CRUD workflows
   - Controls publication status for content
2. Technological literacy:
   - Comfortable with admin dashboards
   - Expects validation, forms, entity relations, and file upload fields

### Portfolio or Exam Evaluator

1. Characteristics:
   - Reviews architecture, API design, code organization, and local deployment
   - Needs clear setup instructions and documentation
2. Technological literacy:
   - Technical reviewer or instructor
   - Expects reproducible local execution through Docker
